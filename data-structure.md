# Структура даних анкети

> **Цей файл містить**: повний перелік полів анкети — логічні імена, типи, ID в Airtable, можливі значення (включно з кодами, що використовує HTML-форма) та відомі неузгодженості між формою і базою.
> **Цей файл не містить**: інструкцій з верстки HTML-форми (`index.html`), кроків AI-обробки фотографій (`image-to-airtable.md`), загального опису архітектури та процесу розробки (`CLAUDE.md`).
>
> Це джерело правди для структури даних. При зміні полів анкети спочатку оновлюється цей файл, потім — `index.html` та `image-to-airtable.md`.

## Опис проекту
Анкета для збору інформації про учасників дослідження (пацієнтів з ампутацією та контрольну групу). Анкета поступово розширюється — реалізація відбувається етапами; повна цільова структура зберігатиметься в Airtable та `index.html` по мірі готовності кожного етапу.

### Наступні етапи (ще не реалізовано)

Повна цільова структура (ампутація, біль, генез, ДОГ ФБіФВ, скринінг сну, ПСІ-батарея, сексуальна функція, об'єктивні дослідження, згода з підписом, ТФБ, психотерапія) описана в технічному документі реєстру (registry_v4-5-7) і буде додаватись частинами — кожна вкладка окремим етапом з оновленням Airtable, `index.html` та цього файлу.

## Пряме збереження з веб-форми в Airtable

`index.html` може записувати анкету напряму в Airtable через REST API (без участі Claude Code):
- Кнопка "Зберегти" запитує пароль, яким зашифрований Airtable API-ключ
- Формат шифрування ключа та процес його встановлення/зміни — `.claude/skills/airtable-key-encryption/SKILL.md`
- Якщо ключ ще не налаштований (порожні константи в `index.html`), форма повідомляє про це і не намагається зберегти
- Перед записом форма шукає в Airtable запис з таким самим Study ID (`Study ID`). Якщо такий запис вже існує — оновлює його (виправлення помилково введених даних), якщо ні — створює новий. Це запобігає появі дублікатів пацієнта при виправленні помилок

## Примітки

- Таблиця налаштована в Airtable та готова до використання
- Нові записи вносяться або напряму з веб-форми (після налаштування шифрованого ключа), або через MCP-підключення (Claude Code) при обробці фото
- При додаванні нових полів структура спочатку оновлюється тут, потім синхронізується з Airtable, `index.html` та `image-to-airtable.md`
