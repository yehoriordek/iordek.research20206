# Журнал змін

> **Цей файл містить**: хронологію змін у проекті по версіях.
> **Цей файл не містить**: поточного опису структури даних, форми чи процесу розробки — для цього дивись `data-structure.md`, `index.html` та `CLAUDE.md` відповідно.

## [0.5.0] — 13 червня 2026 року

### Змінено
- 🚻 **Поле "Стать пацієнта"** — прибрано варіанти "Інша" та "Не вказано", залишено лише "Чоловіча"/"Жіноча" (на екрані та в друкованій версії). Усунено попередню неузгодженість зі значеннями Airtable

## [0.4.0] — 13 червня 2026 року

### Змінено
- 📅 **Поле "Вік пацієнта"** замінено на вибір дати народження (`patient_birthdate`) у веб-формі
  - на екрані поруч з датою народження автоматично відображається кількість повних років пацієнта
  - значення "Вік" для Airtable розраховується автоматично з дати народження
  - у друкованій версії — поле для запису дати народження від руки (дд.мм.рррр)

### Технічні деталі
- Оновлено `data-structure.md`, `index.html` та `image-to-airtable.md` для узгодженого розрахунку віку

## [0.3.0] — 12 червня 2026 року

### Додано
- 📋 Нове поле анкети "Вік пацієнта" (`patient_age`) — додано в таблицю Airtable, форму `index.html`, `data-structure.md` та `image-to-airtable.md`
- 🔐 **index.html** — налаштовано шифрований Airtable API-ключ: пряме збереження анкети з форми в Airtable тепер активне

### Оновлено
- 🖨️ **Режим друку форми**
  - плейсхолдери більше не друкуються на паперовому бланку
  - випадаючі списки в друкованій версії показуються як варіанти для позначення галочкою
- 📋 **Інструкції проєкту**
  - додано правило, що кожен елемент анкети треба одразу продумувати і для екрана, і для паперового заповнення олівцем

### Технічні деталі
- Оновлено `index.html`, `CLAUDE.md` та `image-to-airtable.md` для узгодженого паперового сценарію

## [0.2.2] — 12 червня 2026 року

### Додано
- 🌐 **index.html** — пряме збереження анкети в Airtable через REST API: кнопка "Зберегти" запитує пароль, яким зашифрований Airtable API-ключ, і записує дані без участі Claude Code
- 🔐 **`.claude/skills/airtable-key-encryption/SKILL.md`** — скіл для шифрування/розшифрування Airtable API-ключа (PBKDF2 + AES-256-GCM)

### Оновлено
- 📄 **data-structure.md** — додано опис прямого збереження з форми та посилання на скіл шифрування; уточнено, що поле "Стать" не передається в Airtable для значень `O`/`NA` до узгодження неузгодженості
- 📄 **CLAUDE.md** — оновлено опис поведінки `index.html`; додано критерії, коли можна самостійно комітити/пушити/створювати PR

### Відомо
- ⚠️ Шифрований ключ ще не налаштований (порожні константи в `index.html`) — форма поки показує повідомлення, що збереження не налаштоване

## [0.1.0] — 11 червня 2026 року

### Додано
- ✨ **Пілотна версія проекту** з мінімальною структурою даних
- 📄 **data-structure.md** — описання двох базових полів:
  - Кодовий нікнейм пацієнта (текст)
  - Стать пацієнта (вибір)
- 🌐 **index.html** — веб-форма для збору даних:
  - Режим заповнення (звичайна форма)
  - Режим друку (для друкування та заповнення від руки)
  - Локальне збереження даних (localStorage)
- 🤖 **image-to-airtable.md** — інструкція для AI:
  - Розпізнавання даних з фотографій
  - Контроль якості з уточнюючими запитаннями
  - Запис в Airtable через MCP
- 📋 **CLAUDE.md** — документація проекту та інструкції для розробки

### Технічні деталі
- Форма використовує HTML5 та ванільний JavaScript
- Стилізація адаптована для вебу та друку
- Дані зберігаються локально в localStorage
