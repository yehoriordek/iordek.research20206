# План: перехід анкети на динамічне (повторюване) спостереження

> **Цей файл містить**: узгоджений план переходу від "одного запису на учасника" до моделі з повторюваними вимірами (щоденник болю, контрольні опитування, сесії психотерапії, накопичувальні об'єктивні дослідження). Розбитий на окремі завдання — кожне можна давати в новому чаті як самостійне ТЗ.
> **Цей файл не містить**: фінальної структури полів (буде описана в `data-structure.md` під час виконання кожного завдання), коду форм.
>
> Після виконання всіх завдань цей файл можна видалити (або позначити як завершений) — деталі переїдуть у `data-structure.md` та інші профільні файли.

## Контекст і причина змін

Дослідження передбачає до **8 контактів** з учасником. Потрібно бачити динаміку показників у часі, тому "перезаписування" одного запису не підходить — інформація про попередні візити втрачатиметься.

За результатами обговорення дані розділяються на групи з різною частотою:

1. **Щоденник болю** — 1-3 рази на тиждень: ФБ/ФВ/БК (ВАШ), генез (вірогідно), застосовані техніки, нотатки.
2. **Контрольні опитування** (Сон, ПСІ-батарея, ДОГ ФБіФВ, Сексуальна функція) — рівно 3 рази: Первинне / Всередині циклу / Вихідне.
3. **Сесії психотерапії** — за потреби, накопичувально (як уже є для ТФБ): дата, тип запиту, симптоматика, методи терапії, нотатки.
4. **Об'єктивні дослідження** — накопичувально, у будь-який момент може з'явитись новий запис (неврома, гетеротопічна осифікація тощо), кожен запис зі своєю датою.
5. **Одноразові дані** (без змін, лишаються у "Учасниках"): паспортна частина, ампутація (рівень/третина/RPNI/TMR/дати), скарги, дослідницька та інформована згода. Блок "Біль" (локалізація/характер/патологія кукси) і "Генез" як окремий розділ Етапу 4 — потребують додаткового рішення під час Завдання 1 (генез частково переноситься в щоденник болю).

**Поле "Дата первинного опитування"** (додане раніше) визнано надлишковим — буде видалене (Завдання 0), бо дата першого контрольного опитування фіксуватиметься в таблиці "Опитування" (Завдання 2).

## Завдання 1 — Щоденник болю (нова таблиця + нова форма)

**Обсяг**: середній, аналог уже існуючої схеми "Сесії ТФБ" (`tfb-session.html` + таблиця "Сесії ТФБ").

Нова таблиця Airtable **"Щоденник болю"**, зв'язана з "Учасники" через Study ID (так само, як "Сесії ТФБ"). Кожен запис — один день/сесія:

- **Дата**
- **ФБ — ВАШ** (0-10)
- **ФВ — ВАШ** (0-10)
- **БК — ВАШ** (0-10)
- **Генез (вірогідно)** — вибір ймовірного генезу на цей момент: периферичний / спінальний / церебральний / психокогнітивний / змішаний / невизначено (точний список варіантів узгодити при виконанні)
- **Застосовані техніки** — текстове поле з описом, що робили на сесії
- **Нотатки** — вільний текст

Нова форма `pain-diary.html` (за зразком `tfb-session.html`): пошук учасника за Study ID → форма для одного запису щоденника → збереження створює новий запис у таблиці "Щоденник болю".

**Відкрите питання для узгодження під час виконання**: чи переносити блок "Генез" (Етап 4 поточної анкети — клінічні ознаки 4 типів + тести) повністю в щоденник болю, чи лишити "первинний генез" одноразово в "Учасниках", а в щоденнику фіксувати лише "поточний вірогідний генез" одним полем вибору (схиляюсь до другого варіанту — простіше і відповідає описаній ідеї "Генез вірогідно...").

## Завдання 2 — Контрольні опитування: Сон + ПСІ-батарея + ДОГ + Сексуальна функція (3 етапи)

**Обсяг**: великий — найскладніше завдання, переносить значну частину поточної форми.

Нова таблиця Airtable **"Опитування"**, зв'язана з "Учасники" через Study ID. Кожен запис — одне з трьох контрольних опитувань:

- **Дата**
- **Етап спостереження** — singleSelect: `Первинне` / `Всередині циклу` / `Вихідне`
- Усі поля блоків:
  - Скринінг сну (Етап 6, розділи А-Г + 3 підсумкові поля)
  - ПСІ-батарея: LEC-5, PCL-5, HADS, AIS, DN-4, C-SSRS, DES-T, MoCA (Етапи 7-14)
  - ДОГ ФБіФВ (Етап 5, усі 6 блоків + підсумкові поля)
  - Сексуальна функція (Етап 15)

**Зміни в `index.html`**:
- Перелічені блоки прибираються з основної анкети учасника (або переносяться в окрему форму/режим).
- Потрібна нова форма або новий режим: обрати Study ID + етап спостереження + дату → заповнити блоки → зберегти як новий запис у "Опитуваннях" (без перезапису попередніх).

**Відкрите питання для узгодження**: окрема html-форма (`assessment.html`, за зразком `tfb-session.html`) чи новий екран/вкладка в `index.html` із вибором Study ID та етапу на старті.

## Завдання 3 — Психотерапія як повторювані сесії

**Обсяг**: середній, аналог "Сесії ТФБ".

Нова таблиця Airtable **"Сесії психотерапії"**, зв'язана з "Учасники" через Study ID. Кожен запис:

- **Дата**
- **Тип запиту** (мультивибір — перелік як у поточному блоці "Психотерапія")
- **Симптоматика** (узгодити: текст чи мультивибір)
- **Методи терапії** (узгодити: текст чи мультивибір)
- **Нотатки** (multilineText)

Нова форма `psychotherapy-session.html` (за зразком `tfb-session.html`) з можливістю додавати нові сесії ("Додати сесію" → новий запис).

Поточний блок "Психотерапія" (Етап 19, одне поле "Тип запиту" в "Учасниках") видаляється.

## Завдання 4 — Об'єктивні дослідження як накопичувальний список

**Обсяг**: середній.

Зараз 4 текстові поля (інструментальні дослідження / оперативні втручання / регіонарне знеболення / системне знеболення-фармакотерапія) — кожне довільний текст з усіма записами підряд. Замінити на накопичувальний список записів, кожен зі своєю датою (бо нові знахідки — неврома, ГО тощо — можуть з'являтись у будь-який момент протягом дослідження).

Варіанти структури (узгодити при виконанні):
- Нова таблиця "Об'єктивні дослідження" зі зв'язком до учасника: **Дата**, **Тип** (singleSelect: 4 категорії), **Опис** (multilineText), **Фото** (attachments).
- Або список повторюваних груп полів прямо у формі учасника (без нової таблиці) — простіше, але менш гнучко для додавання "коли завгодно".

Схиляюсь до окремої таблиці (узгоджено з підходом завдань 1 і 3).
