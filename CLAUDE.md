## Архітектура проекту

### Мета
Розробка веб-додатку для збору анкетних даних з подальшим збереженням у базі даних Airtable.

### Основні компоненти

#### 1. **Markdown-файл структури даних** (`data-structure.md`)
- Основний файл проекту
- Містить перелік усіх полів форми
- Описує тип кожного поля, його призначення та можливі значення (для перечислюваних типів)
- Служить джерелом правди для всіх інших компонентів

#### 2. **HTML-форма** (`index.html`)
- Standalone веб-сторінка
- Може відкриватися локально з браузера
- Розміщена на GitHub.io
- Містить дві режими:
  - **Режим заповнення**: для введення даних через форму
  - **Режим друку**: переключач у початку сторінки для превращення форми в друковану версію

#### 3. **AI Skill-файл** (`image-to-airtable.md`)
- Інструкції для моделі Claude для обробки фотографій заповнених від руки анкет
- Містить:
  - Опис структури даних
  - Послідовність кроків для розпізнавання та введення даних
  - Контроль якості з уточнюючими запитаннями
  - Вказівки для запису в Airtable через MCP

#### 4. **Додаткові документи** (за необхідністю)
- `ARCHITECTURE.md` — описання архітектури та взаємодії компонентів
- `UI-GUIDELINES.md` — вказівки щодо інтерфейсу
- `AI-INTERACTION.md` — описання взаємодії користувача з AI при сканування фото
- `CHANGELOG.md` — журнал змін (українська мова)

## Потоки даних

### Спосіб 1: Веб-форма
```
Користувач заповнює HTML-форму → Дані → Airtable (MCP-підключення)
```

### Спосіб 2: Фотографія анкети від руки
```
Користувач робить фото → Надсилає Claude Code → 
AI розпізнає дані → Уточнює через питання → 
Записує в Airtable (MCP)
```

### Спосіб 3: Друкована форма
```
Користувач друкує HTML (режим друку) → Заповнює від руки →
Робить фото → Надсилає Claude Code →処理 як у Способі 2
```

## Принципи розробки

### Архітектура
