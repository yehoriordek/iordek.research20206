# AI Skill: Обробка фотографій анкет та запис в Airtable

> **Цей файл містить**: послідовність дій AI при обробці фотографії заповненої від руки анкети — розпізнавання, уточнення з користувачем, валідація та запис в Airtable.
> **Цей файл не містить**: переліку полів анкети, їх типів, ID в Airtable та можливих значень — це описано в `data-structure.md`, який є джерелом правди. Перед обробкою фото звертайся до нього.

## Мета
Розпізнати дані з фотографії заповненої від руки анкети, уточнити їх якщо потрібно, та записати в базу даних Airtable.

## Особливість друкованої форми

- Друкована версія анкети призначена для заповнення олівцем від руки на папері.
- Тексти-підказки всередині полів не друкуються.
- Елементи вибору в друкованій версії подаються як варіанти для позначення галочкою, а не як випадаючі списки.
- Під час оновлення інструкції або форми треба для кожного поля окремо перевіряти, чи зручно його буде заповнювати від руки на роздрукованому аркуші.

## Структура даних

### Поля для розпізнавання

1. **Кодовий нікнейм пацієнта** (`patient_nickname`)
   - Тип: текст (коротка строка)
   - Приклад: "P001", "RES-002"
   - Обов'язкове: Так

Повний перелік полів, типів та можливих значень — у `data-structure.md`. На момент написання цього файлу анкета містить три поля: кодовий нікнейм пацієнта (`patient_nickname`), стать пацієнта (`patient_gender`) та вік пацієнта (`patient_age`).

## Процес обробки

### Крок 1: Розпізнавання даних з фото
1. Проаналізувати фотографію анкети
2. Розпізнати рукопис та зібрати значення для кожного поля (перелік полів — у `data-structure.md`)
3. Створити попередній список даних

**На цьому кроці не фіксуй непевні значення — переходь до кроку 2.**

### Крок 2: Контроль якості та уточнення

Якщо якесь поле розпізнане з невпевненістю (≤80%), запитай користувача:

**Для кодового нікнейма:**
```
❓ Кодовий нікнейм: Я розпізнав [ЗНАЧЕННЯ]. Це правильно? Якщо ні, напиши правильне значення.
```

**Для статі:**
```
❓ Стать пацієнта: Я розпізнав позначку біля варіанту [ЗНАЧЕННЯ]. Це правильно?
   М — Чоловіча
   Ж — Жіноча
   І — Інша
   Н — Не вказано
   (Напиши букву або слово)
```

**Для віку:**
```
❓ Вік пацієнта: Я розпізнав [ЗНАЧЕННЯ]. Це правильно? Якщо ні, напиши правильне значення.
```

### Крок 3: Обробка відповідей
- Якщо користувач підтвердив — переходь до кроку 4
- Якщо користувач виправив — використай виправлене значення
- Якщо користувач не відповів — запитай ще раз

### Крок 4: Валідація даних

Перед записом перевір:

✅ **Кодовий нікнейм:**
- Не пустий
- Не містить заборонених символів (дозволяються: букви, цифри, дефіс, підкреслення)

✅ **Стать:**
- Є один з варіантів, описаних у `data-structure.md` (HTML-коди `M`, `F`, `O`, `NA`)
- Не пустий
- Якщо розпізнано `O` або `NA` — врахуй неузгодженість зі значеннями Airtable, описану в `data-structure.md`, і узгодь з користувачем, яке значення записати

✅ **Вік:**
- Якщо вказаний — є цілим позитивним числом
- Поле не обов'язкове: якщо на анкеті не вказано вік, просто не записуй це поле

### Крок 5: Запис в Airtable

Використай MCP інструмент для запису нового запису в таблицю "Учасники". База, таблиця та ID полів — у `data-structure.md`.

**Формат запису:**
```json
{
  "Ім'я": "[ЗНАЧЕННЯ_НІКНЕЙМА]",
  "Стать": "[ЗНАЧЕННЯ_СТАТІ]",
  "Вік": [ЗНАЧЕННЯ_ВІКУ]
}
```

Поле "Вік" — не обов'язкове, додавай його тільки якщо значення відоме.

**Приклад:**
```json
{
  "Ім'я": "P001",
  "Стать": "Чоловік",
  "Вік": 34
}
```

### Крок 6: Підтвердження

Повідом користувачу про успіх:

```
✅ Дані успішно записані в базу!
   Нікнейм: [ЗНАЧЕННЯ]
   Стать: [ПЕРЕВЕДЕНЕ_ЗНАЧЕННЯ]
   Вік: [ЗНАЧЕННЯ або "не вказано"]
```

---

## Переведення значень для користувача

При представленні результатів користувачу (HTML-коди → текст):
- `M` → "Чоловіча"
- `F` → "Жіноча"
- `O` → "Інша"
- `NA` → "Не вказано"

---

## Примітки

1. **Якість розпізнавання**: Якщо ти не впевнений у якому-небудь полі, ЗАВЖДИ запитай користувача. Краще запитати один раз, ніж записати неправильні дані.

2. **Обробка помилок**: Якщо користувач вводить неправильне значення для статі, пропоную варіанти ще раз.

3. **Безпека даних**: Нікнейм пацієнта — це кодовий ідентифікатор. Він не повинен містити особисті дані (прізвище, ім'я тощо).

4. **Послідовність**: Запитай спочатку нікнейм, потім стать. Не перепитуй одне поле кілька разів.

5. **Друкований бланк**: Якщо працюєш з фото паперової анкети, очікуй побачити порожні лінії для текстових полів і квадратні місця для позначок біля варіантів відповіді.
