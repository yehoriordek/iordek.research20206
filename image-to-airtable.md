# AI Skill: Обробка фотографій анкет та запис в Airtable

> **Цей файл містить**: послідовність дій AI при обробці фотографії заповненої від руки анкети — розпізнавання, уточнення з користувачем, валідація та запис в Airtable.
> **Цей файл не містить**: переліку полів анкети, їх типів, ID в Airtable та можливих значень — це описано в `data-structure.md`, який є джерелом правди. Перед обробкою фото звертайся до нього.

## Мета
Розпізнати дані з фотографії заповненої від руки анкети, уточнити їх якщо потрібно, та записати в базу даних Airtable.

## Особливість друкованої форми

- Друкована версія анкети призначена для заповнення олівцем від руки на папері.
- Тексти-підказки всередині полів не друкуються.
- Елементи вибору в друкованій версії подаються як варіанти для позначення галочкою, а не як випадаючі списки. Мультивибір — кілька позначених квадратів.
- Toggle-поля (наявність феноменів, ризики, згода) — один квадрат для позначення "так".
- Під час оновлення інструкції або форми треба для кожного поля окремо перевіряти, чи зручно його буде заповнювати від руки на роздрукованому аркуші.

Повний перелік полів, типів та можливих значень — у `data-structure.md`. Станом на цю версію реалізовано Етап 1 (вкладка "Паспорт"), Етап 2 (Ампутація — рівень та деталі), Етап 3 (Біль), Етап 4 (Генез) та Етап 5 (ДОГ ФБіФВ).

Блок "Ампутація" заповнюється на анкеті лише якщо позначено "Є ампутація" у Паспорті. Блоки "Біль", "Генез" та "ДОГ ФБіФВ" — лише якщо у Паспорті позначено хоча б один з феноменів (ФБ/ФВ/БК).

## Процес обробки

### Крок 1: Розпізнавання даних з фото
1. Проаналізувати фотографію анкети
2. Розпізнати рукопис та зібрати значення для кожного поля Етапу 1 — перелік і можливі значення в `data-structure.md`:
   - ПІБ, Study ID, стать, дата народження, військова частина, № палати
   - Тип учасника (Пацієнт/Контроль/Обидва)
   - Наявність феноменів (ампутація / фантомний біль / фантомні відчуття / біль кукси) — позначені квадрати
   - Вживання ПАР, курить, суїцидальний ризик, психіатричний анамнез
   - Симптоматика (мультивибір)
   - Дослідницька згода, пункти згоди, впевненість дослідника
   - Якщо позначено "Є ампутація" — також блок "Ампутація": рівень ампутації та третина для кожної кінцівки (де заповнено), RPNI, тип ампутації, дати поранення/ампутації/госпіталізації/реампутації
   - Для кожного позначеного феномену (ФБ/ФВ/БК) — блок "Біль": локалізація, інтенсивність (VAS 0-10, обведене значення), добовий ритм
   - Характер протікання болю, травматичні спогади, патологія кукси (мультивибір, примітка, джерело виявлення), соматичні реакції на біль
   - Якщо позначено хоча б один феномен (ФБ/ФВ/БК) — також блок "Генез": клінічні ознаки 4 типів генезу (периферичний/церебральний/спінальний/психогенний, мультивибір + поле "інше"), а також позитивні тести та реакції для 3 категорій тестів (церебральні/периферичні/психогенні)
   - Якщо позначено хоча б один феномен (ФБ/ФВ/БК) — також блок "ДОГ ФБіФВ": для кожного з 6 блоків (периферичний/спінальний/церебральний/психокогнітивний генез, предиктор болю в куксі, модулятори) — позначені варіанти (0-4) для 8 питань, дескриптори (мультивибір + "інше"). Підсумуй бали по кожному блоку (0-32) — список питань і варіантів у `data-structure.md`
3. Створити попередній список даних

**На цьому кроці не фіксуй непевні значення — переходь до кроку 2.**

### Крок 2: Контроль якості та уточнення

Якщо якесь поле розпізнане з невпевненістю (≤80%), запитай користувача. Найважливіші поля для уточнення:

**Для Study ID:**
```
❓ Study ID: Я розпізнав [ЗНАЧЕННЯ]. Це правильно? Якщо ні, напиши правильне значення.
```

**Для статі:**
```
❓ Стать пацієнта: Я розпізнав позначку біля варіанту [ЗНАЧЕННЯ]. Це правильно?
   М — Чоловіча
   Ж — Жіноча
   (Напиши букву або слово)
```

**Для дати народження:**
```
❓ Дата народження пацієнта: Я розпізнав [ЗНАЧЕННЯ]. Це правильно? Якщо ні, напиши правильну дату (дд.мм.рррр).
```

**Для решти полів** (ПІБ, в/ч, палата, тип учасника, феномени, ПАР/ризики, симптоматика, згода, ампутація, біль, генез) — якщо позначка нерозбірлива чи поле порожнє, перерахуй знайдені значення і запитай одним повідомленням, чи все правильно, перелічуючи лише сумнівні пункти.

### Крок 3: Обробка відповідей
- Якщо користувач підтвердив — переходь до кроку 4
- Якщо користувач виправив — використай виправлене значення
- Якщо користувач не відповів — запитай ще раз

### Крок 4: Валідація даних

Перед записом перевір:

✅ **Study ID:**
- Не пустий
- Не містить заборонених символів (дозволяються: букви, цифри, дефіс, підкреслення)

✅ **Стать:**
- Є один з варіантів, описаних у `data-structure.md` (HTML-коди `M`, `F`)
- Не пустий

✅ **Дата народження:**
- Якщо вказана — це дійсна дата (дд.мм.рррр), не пізніша за поточну
- Поле не обов'язкове: якщо на анкеті не вказано дату народження, не розраховуй і не записуй поле "Вік"
- Якщо дата вказана — розрахуй повні роки на поточну дату й використай це число як значення поля "Вік" для Airtable

✅ **Вибори (singleSelect)** — тип учасника, вживання ПАР, впевненість дослідника:
- Значення має співпадати з одним із варіантів у `data-structure.md`. Якщо позначка не зроблена — поле не додається до запису

✅ **Мультивибір (multipleSelects)** — симптоматика, пункти згоди:
- Кожне позначене значення має співпадати з варіантами у `data-structure.md`. Якщо нічого не позначено — передай порожній масив

✅ **Toggle/checkbox** — наявність феноменів, курить, суїцидальний ризик, психіатричний анамнез, згода на дослідження, RPNI:
- Позначено = `true`, не позначено = `false`

✅ **Ампутація (Етап 2)** — заповнюється лише якщо позначено "Є ампутація":
- Рівень ампутації для кожної кінцівки — має співпадати з одним із варіантів у `data-structure.md` для відповідної кінцівки (верхньої чи нижньої). Якщо для кінцівки нічого не позначено — поля цієї кінцівки не додаються до запису
- Третина (`в/3`/`1/2`/`н/3`) — додається тільки якщо позначений рівень має поділ на третини і третина позначена
- Дати (поранення/ампутації/госпіталізації/реампутації) — дійсні дати (дд.мм.рррр), не пізніші за поточну; додаються лише якщо заповнені

✅ **Біль (Етап 3)**:
- Локалізація, VAS, добовий ритм — заповнюються тільки для феноменів (ФБ/ФВ/БК), позначених у Паспорті як наявні; для решти не додаються до запису
- VAS — ціле число 0-10 (обведене значення на бланку)
- Добовий ритм, характер протікання болю, патологія кукси, джерело виявлення патології кукси, соматичні реакції на біль — мультивибір; якщо нічого не позначено — передай порожній масив
- Травматичні спогади — один з варіантів `Присутні`/`Відсутні`; якщо не позначено — поле не додається
- "Патологія кукси — джерело виявлення" та "Соматичні реакції на біль" мають у базі варіанти з нетиповим написанням (`Скаржа пацієнта`, `тахікардія`, `М'язова напруження`) — використовуй точні значення з `data-structure.md`, а не виправлені на письмі варіанти

✅ **Генез (Етап 4)** — заповнюється лише якщо позначено хоча б один з феноменів (ФБ/ФВ/БК):
- Клінічні ознаки генезу (периферичні/церебральні/спінальні/психогенні) — мультивибір; якщо нічого не позначено — передай порожній масив. Поля "інше" для кожного типу — додаються лише якщо заповнені
- Тести генезу (церебральні/периферичні/психогенні) — позитивні тести та реакції — мультивибір; якщо нічого не позначено — передай порожній масив
- Декілька варіантів у полях тестів та реакцій мають у базі нетипове написання (`хватальний рефлекс`, `рефлекс захисту`, `телескопія гумкою`/`панчохою`/`муляжем`, `Біль при дефекації/сечовипусканні`, `ФБ земеншився`) — використовуй точні значення з `data-structure.md`, а не виправлені на письмі варіанти

✅ **ДОГ ФБіФВ (Етап 5)** — заповнюється лише якщо позначено хоча б один з феноменів (ФБ/ФВ/БК):
- Для кожного з 6 блоків (периферичний, спінальний, церебральний, психокогнітивний, біль у куксі (предиктор), модулятори) знайди позначений варіант (0-4) для кожного з 8 питань — повний перелік питань і варіантів у `data-structure.md`
- Підсумуй бали по кожному блоку (сума 0-32) — саме сума записується в Airtable, окремі відповіді на питання не зберігаються
- Дескриптори кожного блоку — мультивибір; якщо нічого не позначено — передай порожній масив. Поля "інше" — додаються лише якщо заповнені
- Визнач домінантний генез лише за сумами 4 генез-блоків (периферичний/спінальний/церебральний/психокогнітивний; предиктор і модулятори не враховуються): генез з найбільшою сумою — домінантний; якщо різниця між найбільшою та другою сумою ≤ 2 — запиши "Змішаний (Генез1 / Генез2)"

```json
  "ДОГ — модулятори (сума)": "[0-32]",
  "ДОГ — модулятори (дескриптори)": ["..."],
  "ДОГ — модулятори (інше)": "[ЗНАЧЕННЯ або не додавати, якщо порожнє]",
  "ДОГ — домінантний генез": "[ЗНАЧЕННЯ — назва генезу або 'Змішаний (... / ...)']"
}
```

Поля "Вік", "Тип учасника", "Вживання ПАР", "Впевненість дослідника" — додавай лише якщо значення відоме/позначене. Текстові поля (ПІБ, в/ч, палата) можна додавати порожніми, якщо на анкеті не заповнені. Поля блоку "Ампутація" додаються лише якщо позначено "Є ампутація"; усередині блоку кожне поле — лише якщо для нього є дані (рівень/третина по кожній кінцівці окремо, дати окремо). Поля локалізації/VAS/ритму блоку "Біль" додаються лише для феноменів, позначених у Паспорті як наявні; інші поля блоку "Біль" (характер протікання, травматичні спогади, патологія кукси, соматичні реакції) — завжди, якщо є дані. Поля блоку "Генез" (клінічні ознаки 4 типів + тести) та блоку "ДОГ ФБіФВ" (суми, дескриптори, домінантний генез) додаються лише якщо позначено хоча б один з феноменів (ФБ/ФВ/БК); поля "інше" — лише якщо заповнені.

### Крок 6: Підтвердження

Повідом користувачу про успіх, перелічивши коротко основні дані: Study ID, ПІБ, стать, вік, тип учасника, наявність феноменів, та чи отримана згода на дослідження. Якщо заповнювався блок "Ампутація" — додай короткий підсумок (рівні ампутації по кінцівках). Якщо заповнювався блок "Біль" — додай короткий підсумок (VAS по кожному позначеному феномену). Якщо заповнювався блок "Генез" — додай короткий підсумок (скільки ознак позначено по кожному типу генезу та які тести виявились позитивними). Якщо заповнювався блок "ДОГ ФБіФВ" — додай короткий підсумок (сума балів по кожному з 6 блоків та визначений домінантний генез).

## Примітки

1. **Якість розпізнавання**: Якщо ти не впевнений у якому-небудь полі, ЗАВЖДИ запитай користувача. Краще запитати один раз, ніж записати неправильні дані.

2. **Обробка помилок**: Якщо користувач вводить неправильне значення для статі чи вибору, пропоную варіанти ще раз.

3. **Послідовність**: Запитай спочатку Study ID, потім стать, потім дату народження, далі — решту полів одним блоком уточнюючих питань. Не перепитуй одне поле кілька разів.

4. **Друкований бланк**: Якщо працюєш з фото паперової анкети, очікуй побачити порожні лінії для текстових полів і квадратні місця для позначок біля варіантів відповіді та toggle-полів.
