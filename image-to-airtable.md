# AI Skill: Обробка фотографій анкет та запис в Airtable

## Мета
Розпізнати дані з фотографії заповненої від руки анкети, уточнити їх якщо потрібно, та записати в базу даних Airtable.

## Структура даних

### Поля для розпізнавання

1. **Кодовий нікнейм пацієнта** (`patient_nickname`)
   - Тип: текст (коротка строка)
   - Приклад: "P001", "RES-002"
   - Обов'язкове: Так

2. **Стать пацієнта** (`patient_gender`)
   - Тип: вибір (один варіант)
   - Можливі значення:
     - `M` — Чоловіча
     - `F` — Жіноча
     - `O` — Інша
     - `NA` — Не вказано
   - Обов'язкове: Так

---

## Процес обробки

### Крок 1: Розпізнавання даних з фото
1. Проаналізувати фотографію анкети
2. Розпізнати рукопис та зібрати значення для кожного поля
3. Створити попередній список даних

**На цьому кроці не фіксуй непевні значення — переходь до кроку 2.**

### Крок 2: Контроль якості та уточнення

Якщо якесь поле розпізнане з невпевненістю (≤80%), запитай користувача:

**Для кодового нікнейма:**
```
❓ Кодовий нікнейм: Я розпізнав [ЗНАЧЕННЯ]. Це правильно? Якщо ні, напиши правильне значення.
```

**Для статі:**
```
❓ Стать пацієнта: Я розпізнав [ЗНАЧЕННЯ]. Це правильно?
   М — Чоловіча
   Ж — Жіноча
   І — Інша
   Н — Не вказано
   (Напиши букву або слово)
```

### Крок 3: Обробка відповідей
- Якщо користувач підтвердив — переходь до кроку 4
- Якщо користувач виправив — використай виправлене значення
- Якщо користувач не відповів — запитай ще раз

### Крок 4: Валідація даних

Перед записом перевір:

✅ **Кодовий нікнейм:**
- Не пустий
- Не містить заборонених символів (дозволяються: букви, цифри, дефіс, підкреслення)

✅ **Стать:**
- Є один з варіантів: M, F, O, NA
- Не пустий

### Крок 5: Запис в Airtable

Використай MCP інструмент для запису нового запису в таблицю "Учасники":

**Деталі для запису:**
- **База ID**: `appRtxXIvhSWdb6sJ`
- **Таблиця ID**: `tblMV2RrZGmcGAdqz`
- **Поле Ім'я (ID)**: `fldyUD3kFFZNJmbPF`
- **Поле Стать (ID)**: `fldtIgwWqErKM3Jz0`

**Формат запису:**
```json
{
  "Ім'я": "[ЗНАЧЕННЯ_НІКНЕЙМА]",
  "Стать": "[ЗНАЧЕННЯ_СТАТІ]"
}
```

**Приклад:**
```json
{
  "Ім'я": "P001",
  "Стать": "Чоловік"
}
```

### Крок 6: Підтвердження

Повідом користувачу про успіх:

```
✅ Дані успішно записані в базу!
   Нікнейм: [ЗНАЧЕННЯ]
   Стать: [ПЕРЕВЕДЕНЕ_ЗНАЧЕННЯ]
```

---

## Переведення значень для користувача

При представленні результатів користувачу:
- `M` → "Чоловіча"
- `F` → "Жіноча"
- `O` → "Інша"
- `NA` → "Не вказано"

---

## Примітки

1. **Якість розпізнавання**: Якщо ти не впевнений у якому-небудь полі, ЗАВЖДИ запитай користувача. Краще запитати один раз, ніж записати неправильні дані.

2. **Обробка помилок**: Якщо користувач вводить неправильне значення для статі, пропоную варіанти ще раз.

3. **Безпека даних**: Нікнейм пацієнта — це кодовий ідентифікатор. Він не повинен містити особисті дані (прізвище, ім'я тощо).

4. **Послідовність**: Запитай спочатку нікнейм, потім стать. Не перепитуй одне поле кілька разів.
